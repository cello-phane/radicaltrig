Radical Angle Unit
Forward & Inverse Reference

Complete verified pipeline — Desmos · C · GLSL · Python

§1The One Primitive

Everything in the RAU system is a specialization of a single operation: projecting a point on the unit-square diagonal onto the unit circle. The parameter w measures how far along that diagonal you are. The projection gives (cos, sin) as pure ratios — no transcendental evaluation needed.

w = 0.5 + v·P(v²) ← warp, v = t−½
D = (1−w)² + w² ← always in [½,1]
cos(φ) = (1−w) / √D ← x component
sin(φ) = w / √D ← y component

§2Four Quadrants — the Full Period

The period is 4 RAU = 360°. One unit per quadrant. Integer boundaries land exactly on the four cardinal points. Odd quadrants Q1 and Q3 reverse the warp direction — w → 1−w — which is both the forward rule and the inverse rule.

§3Forward Path φ → (cos, sin)

Three equivalent implementations, all verified to machine precision against each other.

R_vec — the vector cardinal method

Select a cardinal direction vector i from the integer part of φ, then apply the warp and projection within that quadrant. No branching on signs — the cardinal vector carries the sign information.

z = mod(φ, 4)
i = {z<1:(1,0) z<2:(0,1) z<3:(−1,0) z<4:(0,−1)}
w = warp(frac(z)) ← w in [0,1)
D = 1 − 2w + 2w² ← = (1-w)²+w²
cos = f(i, w, D) ← see R_vec formula
sin = g(i, w, D) ← verified: max err 8.3e-16
verified R_vec tested across all four quadrants, 79 points including all cardinals. Max error vs reference: 8.3×10⁻¹⁶ — machine epsilon only. No algorithmic error.

r_cos / r_sin — the offset-branch method

Use m(x, o) = mod(cp·x + o, 4) with offset o=1 for cos, o=0 for sin to route the correct magnitude to the correct output. cp = 1 for RAU input, cp = 2/π for radian input.

r_cos offset Uses m(x, 1) — the +1 shift routes Rs (cos magnitude) into the correct branch at each quadrant boundary.
r_sin offset Uses m(x, 0) — no shift needed, Rc (sin magnitude) appears naturally in the first branch.
bug 2 fix — applies to both methods The guard {x=0: 1, −1} in the integer branch of r_cos must be {mod(x,4)=0: 1, −1} to correctly return cos=+1 at x=4, 8, 12,... as well as x=0.

§4Inverse Path (cos, sin) → φ

Given a circle point R = (cos, sin), recover the original parameter φ. Four methods in increasing precision, all using only the sign bits for quadrant detection.

The odd-quadrant reversal — same rule as forward

Q0, Q2 (even): φ = Q + t ← t counts forward
Q1, Q3 (odd): φ = Q + (1−t) ← t reversed, mirrors forward w→1-w
the critical bug — previously missed Without the (1−t) reversal in Q1 and Q3, the error is up to 1.0 RAU — a full quadrant wrong. The reversal is not optional. It is the exact mirror of the forward warp reversal and must be applied in both directions.

§5Single-Component Recovery

When only sin or only cos is available — one PMOD pin, one ADC channel, one encoder output — w can be recovered algebraically without the other component.

from sin only r_arcsin = |sin² − √(sin²(1−sin²))| / |2sin²−1| Recovers w. Singularity at sin=1/√2 (w=0.5) — use midpoint guard.
exact
from cos only r_arccos = |cos²−1+√(cos²(1−cos²))| / |2cos²−1| Recovers w. Same singularity at cos=1/√2.
exact
real-time strategy — 2-bit sign register Store sign(cos) and sign(sin) as two bits. Update only at zero crossings (infrequent). Between crossings, interpolate w from the single measured component using r_arcsin or r_arccos. No atan2 call at any point in the steady-state loop.
