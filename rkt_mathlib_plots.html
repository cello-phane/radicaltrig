RKT Math Library — Reference

Plots are the function, approx, and error (on the same axes)

1 RAU = π/2 radians. Each approximation below follows the same pattern: reduce the input to a small fixed interval, fit a Remez minimax polynomial on that interval only, then reconstruct exactly using integer/bit arithmetic. Solid lines are reference values, dotted lines are the polynomial, and the red trace (right axis) is absolute error.

§1 — warp(t): the diagonal-to-arc correction

5.6e-7 · float32

Maps the raw diagonal parameter t ∈ [0,1] to the arc-uniform coordinate w = sin(t·π/2) / (sin(t·π/2) + cos(t·π/2)). Every sin/cos call in the library is this polynomial plus one sqrt.

§2 — rau_sincosf vs cs_poly_sincos

1.68e-7 / 1.77e-7

Two independent pipelines for sin/cos. rau_sincosf warps then normalises via one sqrt; cs_poly_sincos fits cos/sin directly with no sqrt at all. Both float32-class.

§3 — erf_poly

3.7e-8 · float32

Degree-10 odd Remez polynomial. EC[0] = 2/√π is recovered independently by the fit. Valid only on [−1.2, 1.2] — diverges outside this domain.

§4 — erfinv_hybrid

7.1e-4 · float16

Polynomial for |x| < 0.949 (94.9% of uniform inputs, zero transcendentals), log+sqrt asymptotic tail beyond. The switch point is visible as a kink in the error trace.

§5 — exp_poly

3.9e-7 · float32

eˣ = 2ⁿ · P(f), with n = ⌊x/ln2⌋ applied exactly via the IEEE-754 exponent field, and P a degree-5 Remez fit on f ∈ [0,1) only.

§6 — ln_opt: corrected vs buggy coefficients

bug fixed this session

LO2, LO3, LO4 were transcription errors in earlier source — off by ~5e-5 in the 6th–7th significant digit. Inflated true error 4.3× (1.24e-5 → 5.34e-5) while staying under the loose float16 pass threshold the whole time.

§7 — arctan tiers: f16 / hsp / precise

3.62e-7 RAU (precise)

Each tier is independently Remez-fit at its target precision — truncating the precise coefficients to a lower bit width would give a worse result than refitting from scratch. The old (pre-Remez) set is ~180× worse than the new precise tier despite using a similar number of terms.
